## 二、Soft Interrupt（immediate steer）

### 目标

用户发 steer 消息时，**立刻**中断当前正在执行的工具 → 注入 steer 消息 → 下个 turn 立即发给 LLM。不等工具自然完成。

比 pi 可做的增强（你说的"可以比它更强"）：
- **中断后的回滚**：pi 不处理回滚。ion 可以让 soft interrupt 触发 file-snapshot 回滚（已写的文件 revert），回到工具执行前的状态。
- **多级粒度**：`interrupt(level: "soft" | "hard" | "abort")` — soft 中断工具但保留上下文、hard 中断并回滚文件、abort 完全停止。

### 实施方案

**在 Agent 加独立的 `interrupted` 信号**（与 `stopped` 分离）：

**(a) 新增字段 — `agent_loop.rs` line 94 附近**
```rust
/// soft interrupt 信号：设 true 后工具执行 select! 立即 break，
/// 但 agent 不退出（继续跑下一 turn），用于 immediate steer。
/// 与 `stopped`（硬停止）独立，每个 turn 用完即重置（对齐 pi 的 interruptController）。
interrupted: std::sync::Arc<std::sync::atomic::AtomicBool>,
```

**(b) 新增方法**
```rust
/// 软中断当前 turn（对齐 pi Agent.interrupt()）
/// 设 interrupted=true，工具 select! 中断，下个 turn 继续
pub fn interrupt(&self) {
    self.interrupted.store(true, Ordering::SeqCst);
    // 唤醒 select! 里的 ticker
}

/// consume-once 检查中断（对齐 pi 换新 controller）
/// 返回 true 表示本轮被 interrupt 过（后续要 drain_steering + continue）
pub fn consume_interrupt(&self) -> bool {
    self.interrupted.swap(false, Ordering::SeqCst)
}
```

**(c) 工具执行 select! 加第 5 分支 — `agent_loop.rs:1049-1094`**
```rust
// 新增第 5 分支：每 200ms tick 检查 interrupted
_ = interrupt_ticker.tick() => {
    if self.interrupted.load(Ordering::SeqCst) {
        tracing::info!("[interrupt] 工具执行中检测到 interrupt，中断当前工具");
        // 工具报 Interrupted 而非 Aborted（可恢复）
        break Err(AgentError::Interrupted);
    }
}
```
（复用现有的 abort_ticker，与 stopped 共享同一 200ms interval tick）

**(d) 工具循环后处理 Interrupted — `agent_loop.rs` 稍后**
```rust
// 在 inner_loop 的 StopReason 匹配里（around line 870-912）
StopReason::Interrupted => {
    // 消耗 interrupt 标志
    self.consume_interrupt();
    // 不返回，drain steering + continue
}
```

**(e) outer_loop 处理 Interrupted**
```rust
// outer_loop 里 match inner_loop 的返回
StopReason::Interrupted => {
    // drain steer 消息
    self.drain_steering().await?;
    // 不退出循环，继续下一轮
}
```

**(f) 中断时正在写的文件回滚（超越 pi）**
如果 `file-snapshot` 扩展启用，在 soft interrupt 时自动触发快照回滚：
```rust
// 中断 → 文件回滚
if let Err(e) = self.extensions.on_gate_check(&messages, ...).await {
    // 触发 file-snapshot restore
}
```
**这个可以后续加，MVP 先只做中断，不做回滚。**

**(g) 工具 Error 映射**
当前 `AgentError` 加 `Interrupted` 变体（和 `Aborted` 分开）：
```rust
pub enum AgentError {
    ...
    Interrupted,  // 新增—工具被 soft interrupt 打断，可恢复
}
```

- ❌ **中断粒度枚举** — MVP 只做 `soft`（中断工具 + 继续 agent），后续再扩展。
