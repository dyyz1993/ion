# UI 无缝体验补全计划

## 目标
补全 5 个后端能力缺口 + 1 个测试场景，让前端 UI 能实现真正的"无缝刷新恢复"。

## 改动清单

### P0: 事件回放（最关键）
**文件**: `src/worker_registry.rs` + `src/bin/ion.rs`

1. WorkerRecord 加字段:
```rust
event_history: VecDeque<serde_json::Value>,  // ring buffer
event_history_cap: usize,  // 默认 200
```

2. stdout reader 转发事件时（worker_registry.rs:577-620）同时 push 到 ring buffer:
```rust
record.event_history.push_back(msg.clone());
if record.event_history.len() > record.event_history_cap {
    record.event_history.pop_front();
}
```

3. subscribe 方法加 replay 参数:
```rust
pub fn subscribe_with_replay(&mut self, worker_id: &str, replay: usize)
    -> (mpsc::Receiver<Value>, Vec<Value>)
```
先返回 ring buffer 最近 N 条，再返回 rx。

4. CLI subscribe 加 `--replay N` 参数（ion.rs:329-339）
5. host socket 的 instance subscribe 分支（ion.rs:3197-3237）支持 replay

### P1: get_session_info
**文件**: `src/agent/agent_loop.rs` + `src/bin/ion_worker.rs`

1. Agent 加字段 + 方法:
```rust
last_stop_reason: Option<StopReason>,
pub fn session_info(&self) -> SessionInfo
```

2. inner_loop 每个 return 点写入 last_stop_reason

3. worker RPC 加 `get_session_info` 分支（合并 model/provider/tokens/queue/turn_index/stop_reason）

4. select! bg_cmd 白名单加入 get_session_info

### P2: agent_stopped 推送
**文件**: `src/bin/ion_worker.rs` + `src/agent/agent_loop.rs`

1. agent.run Ok(()) 后判断 is_stopped():
```rust
if agent.is_stopped() {
    output({"type":"agent_stopped", "reason":"user_abort"})
} else {
    output({"type":"agent_end", "reason":"stop"})
}
```

2. 修 agent_loop.rs:1110-1113: Aborted 路径传 StopReason::Aborted（当前写 Error）

### P3: steer/followUp 队列管理
**文件**: `src/agent/agent_loop.rs` + `src/bin/ion_worker.rs`

1. Agent 加方法:
```rust
pub fn remove_follow_up(&mut self, index: usize) -> Option<Message>
pub fn remove_steering(&mut self, index: usize) -> Option<Message>
```

2. worker RPC 加 `remove_follow_up` / `remove_steering` 分支

3. select! bg_cmd 白名单加入

### P4: 正在跑的 turn 在 list_turns 可见
**文件**: `src/agent/agent_loop.rs` + `src/bin/ion_worker.rs`

1. Agent 加字段:
```rust
current_turn: Option<CurrentTurn>,  // {turn_id, started_at_ms}
```

2. inner_loop turn 开始时设值，结束时清空

3. list_turns 输出前合并 current_turn（status=in_progress）

### P5: 测试
**文件**: `docs/testing/ui_seamless_test.sh`（更新）

1. 加 Case: kill host → 重启 → list_turns 恢复（磁盘 vs 内存）
2. 加 Case: subscribe --replay 50（验证事件回放）
3. 加 Case: get_session_info（验证字段完整）
4. 加 Case: remove_follow_up（验证队列管理）
